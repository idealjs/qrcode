
# idealjs — QR Code Generator (草案需求)

概述
--
这是一个用于生成可自定义样式二维码的小应用。

用户打开页面后，左侧为数据填写/配置面板，右侧为二维码实时预览与导出。

主要特色是生成“圆形”二维码：

在二维码中心保留标准二维码可识别区域，在外圈用无意义的噪点（二维码风格的模块）包裹，使整体呈现圆形外观；

同时支持对三个角上的定位图形（finder patterns）进行自定义形状。

目标

--
- 提供表单化的数据输入（文本/URL/电话等）和样式配置
- 实时预览符合设定样式且可扫码的二维码
- 支持导出 PNG 和 SVG
- 支持自定义三个角的定位图形样式（形状、颜色、大小）

功能需求
--

- UI 布局
	- 左侧：数据与样式配置面板
		- 文本输入（多类型识别：文本/URL/电话/邮件）
		- 大小（像素）/分辨率/边距
		- 前景色/背景色
		- 纠错级别（通俗说明）
			- 低容错 — 只能修复很少的损坏；适合打印清晰且不加装饰的二维码
			- 中等容错 — 常用推荐设置；适合大多数普通场景
			- 高容错 — 可容忍较多遮挡或污损，例如带小 logo 的二维码
			- 极高容错 — 最强容错；用于可能被部分遮挡或需要较大装饰的情况
		- 定位图形（3 个角）的形状选择：方形、圆形、圆角方、SVG 自定义
		- 外圈噪点：开/关、密度、随机种子
	- 右侧：实时预览区域（可切换 SVG/Canvas 视图）及导出按钮

- 圆形外观实现要点
	- 中心区域：使用标准二维码生成方式（确保扫描可靠性）
	- 外圈：在标准二维码外部包裹一圈“无意义”的二维码风格模块或点阵，使整体视觉为圆形；这些噪点应不破坏中心二维码可读性
	- 可选方案：生成比目标尺寸略大的二维码，然后对外层模块按圆形遮罩或替换为随机噪点；或在渲染层叠加一层独立噪点图层

- 定位图形（finder patterns）自定义
	- 允许为三个角分别设置形状（一致或独立）、颜色和尺寸
	- 提供内建形状（方块、圆、圆角方）并允许用户上传自定义 SVG 以替换默认定位图形

- 导出与兼容性
	- 导出 PNG（高分辨率）和 SVG（保真可编辑）
	- 在常见扫码器上可正常识别（需通过验收测试）

实现选项（技术路线）
--

- 可使用现成二维码库生成基础矩阵（例如 `qrcode-generator`、`qrcode`、`qr-code-styling` 等），由库负责编码与纠错逻辑
- 渲染层：可使用 Canvas 或 SVG 实现可定制渲染；SVG 输出便于导出与替换定位图形
- 圆形效果实现思路：对模块进行分层处理——先绘制中心标准二维码，再在外圈绘制噪点层或对外层模块按圆形掩模处理
- 定位图形渲染：在生成矩阵后，识别三角定位区域并使用所选形状替换默认样式

验收标准
--

- 页面加载后可见左侧表单与右侧预览，两部分在宽屏下左右展示，窄屏下垂直堆叠
- 输入数据并修改样式后预览实时更新（延迟 < 200 ms）
- 导出的 PNG/SVG 在主流扫码 App（微信/手机相机/第三方扫码）上可识别
- 自定义的定位图形能正确渲染且不影响可识别性

简单 UI 草图
--

- 左栏（表单）
	- 数据输入
	- 样式配置
	- 定位图形配置（角 1 / 2 / 3）
	- 导出按钮（PNG / SVG）
- 右栏（预览）
	- 实时渲染画布或 SVG
	- 下载按钮、复制图片到剪贴板
